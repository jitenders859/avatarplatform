React SDK

Integrate the AvatarPlatform talking agent into any React application with a single component. Works with Next.js, Vite, Remix, and Create React App.

Installation

npm install @avatar-platform/react
# or
yarn add @avatar-platform/react
# or
pnpm add @avatar-platform/react
ℹ️

The React SDK wraps lipsync-sdk.js and dynamically injects it into the page. No bundler configuration is needed. React 17+ and React 18 are both supported.

AvatarWidget

The primary component. Drop it anywhere in your component tree — typically near the root layout so it persists across route changes.

import { AvatarWidget } from '@avatar-platform/react';

export default function App() {
  return (
    <>
      <YourApp />
      <AvatarWidget
        botId="YOUR_BOT_ID"
        position="bottom-right"
        theme="#6366f1"
        size="md"
        startOpen={false}
        compactMobile
      />
    </>
  );
}

Props

PropTypeDefaultDescription
botId string — Required. The public ID of your project. Find it in the dashboard → Embed tab.
position 'bottom-right' | 'bottom-left' 'bottom-right' Position of the floating launcher button.
theme string '#7c6af5' Accent colour for the widget UI. Any valid CSS hex colour.
size 'sm' | 'md' | 'lg' 'md' Size of the avatar character in the open panel.
startOpen boolean false If true, the panel opens automatically on load.
compactMobile boolean true Hides the avatar character on screens narrower than 480px to save space.
offsetX number 0 Horizontal offset in pixels from the edge of the screen.
offsetY number 0 Vertical offset in pixels from the bottom of the screen.

useAvatarPlatform hook

Access the widget API programmatically — ask questions, listen for answers, or control the open/close state from anywhere in your app.

import { AvatarWidget, useAvatarPlatform } from '@avatar-platform/react';

function SupportButton() {
  const { ask, open, close } = useAvatarPlatform('YOUR_BOT_ID');

  const handleSearch = async (query: string) => {
    const result = await ask(query);
    console.log(result.answer);   // AI answer string
    console.log(result.sources);  // source files used
    open();                        // open the widget to show the reply
  };

  return (
    <button onClick={() => handleSearch('How do I reset my password?')}>
      Ask the AI
    </button>
  );
}

Hook return values

MethodSignatureDescription
ask (question: string, sessionId?: string) => Promise<AskResult> Send a question to the agent. Returns { answer, sources, sessionId }. Does not require voice or the widget to be open.
preload (botId: string) => Promise<Config> Pre-fetch the bot configuration to warm the browser cache before the widget renders.
open () => void Open the chat panel programmatically.
close () => void Close the chat panel.
hide () => void Hide the launcher button and panel entirely.
show () => void Show the launcher button after hiding it.

TypeScript types

interface AvatarWidgetProps {
  botId: string;
  position?: 'bottom-right' | 'bottom-left';
  theme?: string;
  size?: 'sm' | 'md' | 'lg';
  startOpen?: boolean;
  compactMobile?: boolean;
  offsetX?: number;
  offsetY?: number;
}

interface AskResult {
  answer: string;
  sources: {
    title: string;
    url: string | null;
    snippet: string;
  }[];
  sessionId: string;
}

interface UseAvatarPlatformReturn {
  ask:     (question: string, sessionId?: string) => Promise<AskResult>;
  preload: (botId: string) => Promise<unknown>;
  open:    () => void;
  close:   () => void;
  hide:    () => void;
  show:    () => void;
}

Next.js example

For Next.js App Router, mark the component as a client component and add the widget to your root layout:

// app/layout.tsx
'use client';
import { AvatarWidget } from '@avatar-platform/react';

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        {children}
        <AvatarWidget botId={process.env.NEXT_PUBLIC_BOT_ID!} position="bottom-right" />
      </body>
    </html>
  );
}
⚠️

SSR note: The widget renders on the client only. It is safe to use inside 'use client' components. The component tree above it can remain a Server Component.
